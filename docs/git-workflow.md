# Git Workflow

## Branches

- `main`: stable source of truth
- `data/*`: data updates
- `docs/*`: documentation updates
- `feature/*`: web app features

## Commit examples

- `docs: add data policy`
- `data: add nrw holidays v0.3`
- `data: update official source inventory`
- `chore: initialize repository`
- `feat: add calendar generation script`

## Pull request checklist

- Source URL included
- lastCheckedAt updated
- status set
- data scope policy followed
- no school-specific dates in default dataset
